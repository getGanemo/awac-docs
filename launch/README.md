# AWaC launch — drafts and assets

This folder holds drafts, lists, and operational checklists for the AWaC public launch (target: when v1.0.0 ships).

> **All content here is a draft.** Fernando publishes everything manually — never auto-post via tools, never bulk-DM the people list, never share this folder publicly until launch day.

## Files

| File | Purpose |
|---|---|
| `show-hn.md` | Three Show HN draft variants (problem-first / solution-first / gap). Hooks for A/B and follow-up plan. |
| `twitter-thread.md` | 8-tweet primary thread (EN), Spanish version, hook alternatives, follow-up plan. |
| `linkedin-post.md` | LinkedIn long + short versions (ES — primary audience LATAM/Spain). |
| `reddit-r-claudeai.md` | r/ClaudeAI post + anticipated comments. |
| `reddit-r-cursor.md` | r/cursor post + anticipated comments. |
| `dev-to-blog.md` | Long-form blog (1500+ words). Cross-post to Hashnode + Medium with canonical URL. |
| `people-to-ping.md` | 20 individuals to reach out to **after** launch posts go live, by tier. |
| `metrics.md` | Baseline metrics + targets + checkpoints (day 0 / week 1 / month 1 / quarter 1). |
| `babysit.md` | 2-week post-launch maintenance checklist. Issue triage flow, PR flow, reply templates. |

## Hard rules during launch

- ❌ Never paste the same DM twice. Customize first sentence per recipient.
- ❌ Never bulk-message the 20-people list.
- ❌ Never publish content from this folder without Fernando's explicit go.
- ❌ Never skip the `dev.to / Hashnode / Medium` `canonical_url` field — duplicate content tanks SEO.

## Update protocol

Update `metrics.md` continuously over the launch + first month. Other files in this folder are reference; `metrics.md` is the live document.
